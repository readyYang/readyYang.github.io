[ACTF新生赛2020]SoulLike

按照惯例先分析程序：64位无壳

放入64位IDA中，直接查看main函数

逻辑清晰，简单易懂，进入sub_83A函数，失败了，出现了too big function报错，有种不好的预感，上网找解决方法，大佬说：

修改配置文件：【IDA 安装目录】\cfg\hexrays.cfg
源文件为：
MAX_FUNCSIZE = 64 // Functions over 64K are not decompiled

修改为

MAX_FUNCSIZE = 1024 // Functions over 64K are not decompiled

重新启动IDA，使配置生效。
进入sub_83A函数的时候加载了很久，不好的预感更强烈了

进去后发现都是这样的简单加密，但是有整整3000行

最后是普通而又简单的字符串对比，3000行代码，手撸是不可能了，得想帮法解决

我的思路是写一个脚本，它能把我的输入按行逆序输出，然后我直接复制黏贴再用编译器自带的查找替换功能把++全部变成--，再跑一遍代码就可以得到flag了，因为是要逆序，我是尝试用递归写的脚本，但是能力有限，跑不起来，用AI写的递归脚本也跑不了，好像是为了防止栈溢出，所以对于递归函数，编译器的输入是有限制的，根本到不了3000行，综上所述我后来放弃了递归，用AI写的迭代脚本

将复制的代码黏贴进去得到按行逆序输出的代码后就可以查找替换跑出flag了，脚本太长就不写在这了

写出这道题后，我想看看别的师傅有没有别的方法，就去看了一圈，发现他们全部都是爆破出的脚本，下面是一个师傅的爆破脚本

总结：这道题用的方法与别的师傅都不一样，对我来说，这题告诉我该提升的还是代码的编写能力，不能太依靠AI
